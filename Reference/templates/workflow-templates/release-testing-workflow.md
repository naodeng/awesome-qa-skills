# 发布测试工作流

## 工作流概述

**适用场景**: 产品版本发布前的完整测试流程  
**涉及 Skills**: test-strategy, regression-testing, performance-testing, security-testing, test-reporting  
**预计时间**: 3-7 天

---

## 发布测试时间线

```
发布准备（T-7）
    ↓
功能冻结（T-5）
    ↓
回归测试（T-4 到 T-2）
    ↓
发布候选版本（T-1）
    ↓
Go/No-Go 决策（T-1）
    ↓
正式发布（T）
    ↓
发布后监控（T+1 到 T+7）
```

---

## 阶段1: 发布准备（T-7）

### 1.1 发布测试计划

**使用 Skill**: `test-strategy`

**活动**:
```markdown
1. 确定发布范围和目标
2. 识别测试重点和风险
3. 制定测试策略和计划
4. 分配测试资源和时间
```

**发布检查清单**:
```markdown
## 发布信息
- [ ] 发布版本号已确定
- [ ] 发布日期已确定
- [ ] 发布范围已明确
- [ ] 发布目标已对齐

## 测试准备
- [ ] 测试环境已准备
- [ ] 测试数据已准备
- [ ] 测试工具已就绪
- [ ] 测试团队已到位

## 风险评估
- [ ] 技术风险已识别
- [ ] 业务风险已评估
- [ ] 缓解措施已制定
- [ ] 应急预案已准备

## 沟通协调
- [ ] 发布计划已同步
- [ ] 干系人已通知
- [ ] 沟通渠道已建立
- [ ] 升级机制已明确
```

**输出**:
- 发布测试计划
- 测试时间表
- 风险评估报告

## 阶段2: 功能冻结（T-5）

### 2.1 代码冻结确认

**检查项**:
```markdown
- [ ] 所有计划功能已完成
- [ ] 代码已合并到发布分支
- [ ] 构建版本已生成
- [ ] 版本号已标记
```

### 2.2 冒烟测试

**活动**:
```markdown
1. 部署到测试环境
2. 执行冒烟测试套件
3. 验证核心功能
4. 确认环境稳定
```

**冒烟测试检查清单**:
```markdown
- [ ] 应用启动正常
- [ ] 登录功能正常
- [ ] 核心业务流程可用
- [ ] 关键集成点正常
- [ ] 数据库连接正常
- [ ] 第三方服务正常
```

## 阶段3: 回归测试（T-4 到 T-2）

### 3.1 完整回归测试

**使用 Skill**: `regression-testing`

**测试范围**:
```markdown
## 功能测试
- [ ] 核心业务流程（100%）
- [ ] 新增功能（100%）
- [ ] 变更影响区域（100%）
- [ ] 一般功能（80%）
- [ ] 低频功能（50%）

## 集成测试
- [ ] 内部模块集成
- [ ] 第三方服务集成
- [ ] 数据库集成
- [ ] API 集成

## 兼容性测试
- [ ] 浏览器兼容性
- [ ] 操作系统兼容性
- [ ] 设备兼容性
- [ ] 版本兼容性
```

### 3.2 性能测试

**使用 Skill**: `performance-testing`

**测试类型**:
```markdown
1. 负载测试
   - 目标: 验证系统在预期负载下的表现
   - 指标: 响应时间、吞吐量、资源使用

2. 压力测试
   - 目标: 找到系统的极限
   - 指标: 最大并发、崩溃点、恢复能力

3. 稳定性测试
   - 目标: 验证长时间运行的稳定性
   - 指标: 内存泄漏、性能衰减、错误率
```

**性能指标**:
| 指标 | 目标 | 可接受 | 不可接受 |
|------|------|--------|----------|
| 响应时间 | <200ms | <500ms | >1s |
| 吞吐量 | >1000 TPS | >500 TPS | <500 TPS |
| 错误率 | <0.1% | <1% | >1% |
| CPU 使用率 | <60% | <80% | >80% |
| 内存使用率 | <70% | <85% | >85% |

### 3.3 安全测试

**使用 Skill**: `security-testing`

**测试项**:
```markdown
## 认证和授权
- [ ] 登录安全
- [ ] 会话管理
- [ ] 权限控制
- [ ] 密码策略

## 输入验证
- [ ] SQL 注入
- [ ] XSS 攻击
- [ ] CSRF 攻击
- [ ] 文件上传安全

## 数据安全
- [ ] 敏感数据加密
- [ ] 数据传输安全
- [ ] 数据存储安全
- [ ] 日志安全

## 配置安全
- [ ] 默认配置安全
- [ ] 错误信息泄露
- [ ] 安全头配置
- [ ] 依赖漏洞扫描
```

## 阶段4: 发布候选版本（T-1）

### 4.1 最终验证

**验证检查清单**:
```markdown
## 功能验证
- [ ] 所有测试用例已执行
- [ ] 测试通过率 >95%
- [ ] 无 P0/P1 缺陷
- [ ] P2 缺陷已评估

## 性能验证
- [ ] 性能测试通过
- [ ] 性能指标达标
- [ ] 无性能回归
- [ ] 资源使用合理

## 安全验证
- [ ] 安全扫描通过
- [ ] 无高危漏洞
- [ ] 安全配置正确
- [ ] 合规性检查通过

## 文档验证
- [ ] 发布说明已完成
- [ ] 用户文档已更新
- [ ] API 文档已更新
- [ ] 已知问题已记录
```

### 4.2 Go/No-Go 决策

**Go/No-Go 决策矩阵**:

| 维度 | Go 标准 | No-Go 标准 |
|------|---------|------------|
| 功能完整性 | 100% 核心功能可用 | 核心功能不可用 |
| 缺陷情况 | 无 P0/P1 缺陷 | 存在 P0/P1 缺陷 |
| 性能指标 | 所有指标达标 | 关键指标不达标 |
| 安全性 | 无高危漏洞 | 存在高危漏洞 |
| 测试覆盖 | >90% | <80% |
| 团队信心 | 高 | 低 |

**决策会议议程**:
```markdown
1. 测试总结报告（15分钟）
2. 质量指标评审（10分钟）
3. 风险评估（10分钟）
4. 已知问题讨论（10分钟）
5. Go/No-Go 投票（5分钟）
6. 行动计划（10分钟）
```

**输出**:
- Go/No-Go 决策
- 发布风险报告
- 应急预案

## 阶段5: 正式发布（T）

### 5.1 发布执行

**发布检查清单**:
```markdown
## 发布前
- [ ] 备份已完成
- [ ] 回滚计划已准备
- [ ] 监控已就绪
- [ ] 团队已待命

## 发布中
- [ ] 按发布计划执行
- [ ] 实时监控系统状态
- [ ] 记录发布过程
- [ ] 及时沟通进展

## 发布后
- [ ] 冒烟测试通过
- [ ] 监控指标正常
- [ ] 用户反馈正常
- [ ] 发布完成通知
```

### 5.2 发布验证

**验证项**:
```markdown
- [ ] 版本号正确
- [ ] 核心功能正常
- [ ] 性能指标正常
- [ ] 无明显错误
- [ ] 用户可正常访问
```

## 阶段6: 发布后监控（T+1 到 T+7）

### 6.1 持续监控

**监控指标**:
```markdown
## 系统指标
- CPU 使用率
- 内存使用率
- 磁盘使用率
- 网络流量

## 应用指标
- 响应时间
- 错误率
- 请求量
- 并发用户数

## 业务指标
- 用户活跃度
- 转化率
- 关键业务指标
- 用户反馈
```

### 6.2 问题响应

**问题分级**:
| 级别 | 定义 | 响应时间 | 处理策略 |
|------|------|----------|----------|
| P0 | 系统不可用 | 立即 | 紧急修复或回滚 |
| P1 | 核心功能异常 | 1小时 | 优先修复 |
| P2 | 一般功能异常 | 4小时 | 计划修复 |
| P3 | 轻微问题 | 1天 | 下版本修复 |

**发布报告模板**:

```markdown
- 执行用例: [数量]
```

```markdown
3. [步骤 3] - 负责人: [姓名] - 预计时间: [X分钟]

## 回滚验证
- [ ] 系统启动正常
- [ ] 核心功能可用
- [ ] 数据完整性验证
- [ ] 性能指标正常

## 沟通计划
- [ ] 通知干系人
- [ ] 更新状态页面
- [ ] 用户通知
- [ ] 事后分析
```

---

## 最佳实践

1. **充分准备**: 提前规划，预留缓冲时间
2. **全面测试**: 不遗漏任何测试类型
3. **风险管理**: 识别风险，制定缓解措施
4. **清晰沟通**: 及时同步进展和问题
5. **应急准备**: 准备回滚计划和应急预案
