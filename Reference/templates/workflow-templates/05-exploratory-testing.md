# 探索性测试工作流

## 工作流概述

**适用场景**: 快速探索和发现未知问题  
**涉及 Skills**: manual-testing, bug-reporting, test-case-writing  
**预计时间**: 2-4 小时/会话

---

## 工作流步骤

### 1. 测试章程制定

**使用 Skill**: `manual-testing`

**输入**:
- 测试目标
- 时间限制
- 测试范围

**活动**:
```markdown
1. 定义测试任务和目标
2. 确定测试时间盒（通常 90-120 分钟）
3. 识别测试区域和重点
4. 准备测试环境和数据
```

**测试章程模板**:
```
探索: [测试区域]
使用: [测试工具/方法]
发现: [期望发现的问题类型]
时间: [时间盒]
```

**输出**:
- 测试章程
- 测试准备清单

---

### 2. 探索性测试执行

**使用 Skill**: `manual-testing`

**输入**:
- 测试章程
- 测试环境

**活动**:
```markdown
1. 按照测试章程开始探索
2. 应用启发式测试方法
3. 记录测试路径和发现
4. 深入调查可疑行为
```

**启发式方法（SFDPOT）**:
- **S**tructure: 测试结构和架构
- **F**unction: 测试功能和特性
- **D**ata: 测试数据处理
- **P**latform: 测试平台兼容性
- **O**perations: 测试操作流程
- **T**ime: 测试时间相关行为

**输出**:
- 测试笔记
- 发现的问题
- 测试路径记录

---

### 3. 问题记录和报告

**使用 Skill**: `bug-reporting`

**输入**:
- 发现的问题
- 测试笔记

**活动**:
```markdown
1. 确认问题可复现
2. 简化复现步骤
3. 评估问题严重程度
4. 编写详细的 Bug 报告
```

**输出**:
- Bug 报告
- 问题清单

---

### 4. 测试总结和用例提取

**使用 Skill**: `test-case-writing`

**输入**:
- 测试笔记
- 发现的问题

**活动**:
```markdown
1. 总结测试会话
2. 提取有价值的测试场景
3. 转化为正式测试用例
4. 更新测试用例库
```

**输出**:
- 测试会话报告
- 新增测试用例
- 测试改进建议

## 探索性测试章程示例

### 示例 1: 登录功能探索
```
探索: 用户登录功能的安全性和边界情况
使用: Chrome 浏览器、开发者工具、不同类型的测试账号
发现: 安全漏洞、输入验证问题、错误处理缺陷
时间: 90 分钟
```

### 示例 2: 支付流程探索
```
探索: 支付流程在网络异常情况下的行为
使用: 网络限速工具、不同支付方式、模拟网络中断
发现: 数据一致性问题、超时处理、用户体验问题
时间: 120 分钟
```

### 示例 3: 移动端兼容性探索
```
探索: 应用在不同设备和系统版本上的表现
使用: 真机测试、模拟器、不同屏幕尺寸
发现: 布局问题、性能问题、兼容性问题
时间: 90 分钟
```

## 探索性测试技巧

### 1. 测试旅程（Test Tours）

**功能旅程**: 按功能模块逐一探索  
**数据旅程**: 关注数据流转和处理  
**用户旅程**: 模拟真实用户使用场景  
**边界旅程**: 专注测试边界和极限情况  
**错误旅程**: 故意触发错误和异常

### 2. 启发式测试方法

**CRUD**: Create, Read, Update, Delete  
**RCRCRC**: Recent, Core, Risk, Configuration, Repaired, Chronic  
**FCC CUTS VIDS**: File, Configuration, Compatibility, User, Time, Security, Volume, Input, Data, Structure

### 3. 测试思维模式

- **好奇心**: 保持好奇，问"如果...会怎样？"
- **怀疑论**: 质疑假设，寻找反例
- **创造力**: 尝试非常规操作
- **系统性**: 有条理地探索，避免遗漏

## 最佳实践

1. **时间盒管理**: 严格控制测试时间，避免过度探索
2. **实时记录**: 边测试边记录，不要事后回忆
3. **结对测试**: 两人协作，一人测试一人记录
4. **多样化**: 使用不同的测试方法和工具
5. **知识分享**: 及时分享发现和经验

## 常见问题

**Q: 探索性测试和脚本化测试的区别？**  
A: 探索性测试强调学习和发现，脚本化测试强调验证和回归。

**Q: 什么时候做探索性测试？**  
A: 新功能上线前、重大变更后、发现问题集中的区域。

**Q: 如何评估探索性测试的效果？**  
A: 看发现的问题数量和质量、测试覆盖的广度和深度。

**Q: 探索性测试需要什么技能？**  
A: 好奇心、批判性思维、领域知识、测试经验。
