# 安全测试工作流 | Security Testing Workflow

**版本**: 1.0.0  
**最后更新**: 2026-02-09  
**适用场景**: 安全漏洞扫描、渗透测试、安全合规检查

---

## 工作流概述

安全测试工作流用于识别和验证系统的安全漏洞，确保应用符合安全标准和最佳实践。

### 适用场景
- 新版本发布前的安全验证
- 定期安全审计
- 合规性检查（OWASP、PCI-DSS、GDPR）
- 渗透测试
- 安全漏洞修复验证

### 涉及的 Skills
- `test-strategy` - 安全测试策略
- `security-testing` - 安全测试执行
- `test-reporting` - 安全测试报告

---

## 工作流步骤

### 阶段 1: 安全测试规划

#### 1.1 确定测试范围
- [ ] 识别测试目标（Web、API、移动端）
- [ ] 确定测试类型（漏洞扫描、渗透测试、代码审计）
- [ ] 定义安全标准（OWASP Top 10、CWE）
- [ ] 评估风险等级

**使用 Skill**: `test-strategy`

**提示词示例**:
```
请为以下系统制定安全测试策略：

系统类型：[Web 应用 / API 服务 / 移动应用]
技术栈：[列出主要技术]
敏感数据：[用户信息、支付数据等]
合规要求：[GDPR、PCI-DSS 等]

请输出：
1. 安全测试范围
2. 测试优先级
3. 测试方法选择
4. 资源和时间评估
```

#### 1.2 准备测试环境
- [ ] 搭建测试环境
- [ ] 配置安全测试工具
- [ ] 获取测试授权
- [ ] 准备测试账号和数据

---

### 阶段 2: 漏洞扫描

#### 2.1 自动化漏洞扫描
- [ ] 使用 OWASP ZAP / Burp Suite 扫描
- [ ] 执行 SQL 注入测试
- [ ] 执行 XSS 测试
- [ ] 执行 CSRF 测试
- [ ] 检查敏感信息泄露

**使用 Skill**: `security-testing`

**提示词示例**:
```
请为以下 Web 应用设计安全测试用例：

应用 URL：https://example.com
主要功能：用户注册、登录、支付
技术栈：React + Node.js + MySQL

请覆盖 OWASP Top 10：
1. 注入攻击（SQL、NoSQL、命令注入）
2. 失效的身份认证
3. 敏感数据泄露
4. XML 外部实体（XXE）
5. 失效的访问控制
6. 安全配置错误
7. 跨站脚本（XSS）
8. 不安全的反序列化
9. 使用含有已知漏洞的组件
10. 不足的日志记录和监控

请输出详细的测试用例和验证步骤。
```

#### 2.2 手动安全测试
- [ ] 业务逻辑漏洞测试
- [ ] 权限绕过测试
- [ ] 会话管理测试
- [ ] 文件上传安全测试
- [ ] API 安全测试

**常见测试点**:
- 认证和授权
- 会话管理
- 输入验证
- 输出编码
- 加密存储
- 错误处理
- 日志记录

---

### 阶段 3: 渗透测试

#### 3.1 信息收集
- [ ] 域名和 IP 信息
- [ ] 端口扫描
- [ ] 服务识别
- [ ] 目录枚举
- [ ] 技术栈识别

#### 3.2 漏洞利用
- [ ] 尝试已知漏洞利用
- [ ] 测试弱密码
- [ ] 测试默认配置
- [ ] 社会工程学测试（如授权）

#### 3.3 权限提升
- [ ] 测试水平权限提升
- [ ] 测试垂直权限提升
- [ ] 测试特权功能访问

---

### 阶段 4: 结果分析和报告

#### 4.1 漏洞分类和评级
- [ ] 按严重程度分类（严重、高、中、低）
- [ ] 评估业务影响
- [ ] 确定修复优先级
- [ ] 提供修复建议

**漏洞严重程度**:
- **严重**: 可直接获取系统控制权或敏感数据
- **高**: 可能导致重要数据泄露或系统破坏
- **中**: 可能导致部分功能受损或信息泄露
- **低**: 影响较小，但仍需修复

#### 4.2 生成安全测试报告
- [ ] 执行摘要
- [ ] 漏洞详细描述
- [ ] 复现步骤
- [ ] 修复建议
- [ ] 风险评估

**使用 Skill**: `test-reporting`

---

### 阶段 5: 修复验证

#### 5.1 漏洞修复
- [ ] 开发团队修复漏洞
- [ ] 代码审查
- [ ] 安全配置更新

#### 5.2 回归测试
- [ ] 验证漏洞已修复
- [ ] 确认无新漏洞引入
- [ ] 更新安全基线

## 安全测试工具

### OWASP ZAP
- **用途**: Web 应用漏洞扫描
- **示例**: `skills/testing-types/security-testing/examples/owasp-zap-scan`

### Burp Suite
- **用途**: Web 应用渗透测试
- **特点**: 功能强大、插件丰富

### Nmap
- **用途**: 端口扫描、服务识别
- **特点**: 网络安全审计标准工具

### SQLMap
- **用途**: SQL 注入自动化测试
- **特点**: 支持多种数据库

## OWASP Top 10 检查清单

- [ ] A01:2021 - 失效的访问控制
- [ ] A02:2021 - 加密机制失效
- [ ] A03:2021 - 注入
- [ ] A04:2021 - 不安全设计
- [ ] A05:2021 - 安全配置错误
- [ ] A06:2021 - 易受攻击和过时的组件
- [ ] A07:2021 - 身份识别和身份验证失败
- [ ] A08:2021 - 软件和数据完整性故障
- [ ] A09:2021 - 安全日志和监控失败
- [ ] A10:2021 - 服务器端请求伪造（SSRF）

---

## 最佳实践

1. **定期测试**: 每次发布前进行安全测试
2. **自动化扫描**: 集成到 CI/CD 流程
3. **及时修复**: 高危漏洞优先修复
4. **安全培训**: 提升团队安全意识
5. **持续监控**: 部署后持续监控安全事件
