## Schema 定义

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "testSuite": {
      "type": "object",
      "required": ["id", "name", "testCases"],
      "properties": {
        "id": { "type": "string" },
        "name": { "type": "string" },
        "description": { "type": "string" },
        "version": { "type": "string" },
        "created": { "type": "string", "format": "date-time" },
        "updated": { "type": "string", "format": "date-time" },
        "author": { "type": "string" },
        "tags": { "type": "array", "items": { "type": "string" } },
        "testCases": {
          "type": "array",
          "items": {
            "type": "object",
            "required": ["id", "title", "priority", "category", "status"],
            "properties": {
              "id": { "type": "string" },
              "title": { "type": "string" },
              "description": { "type": "string" },
              "priority": { "enum": ["critical", "high", "medium", "low"] },
              "category": { "enum": ["functional", "negative", "security", "performance", "ui", "integration"] },
              "status": { "enum": ["not_run", "pass", "fail", "blocked", "skip"] },
              "assignedTo": { "type": "string" },
              "tags": { "type": "array", "items": { "type": "string" } },
              "estimatedTime": { "type": "integer" },
              "automationStatus": { "enum": ["manual", "automated", "in_progress", "not_feasible"] },
              "automationScript": { "type": "string" }
            }
          }
        }
      }
    }
  }
}
```

## 使用示例

### 1. 导入到测试框架
```javascript
// Jest/Vitest
import testSuite from './test-cases.json';

describe(testSuite.testSuite.name, () => {
  testSuite.testSuite.testCases.forEach(testCase => {
    test(testCase.title, async () => {
      // 执行测试步骤
      for (const step of testCase.testSteps) {
        await executeStep(step);
      }
      
      // 验证预期结果
      for (const result of testCase.expectedResults) {
        await verifyResult(result);
      }
    });
  });
});
```

### 2. API 集成
```bash
# 上传测试用例到测试管理系统
curl -X POST https://testmanagement.example.com/api/v1/test-cases \
  -H "Content-Type: application/json" \
  -d @test-cases.json
```

### 3. 生成报告
```javascript
const testSuite = require('./test-cases.json');
const stats = testSuite.testSuite.statistics;

console.log(`通过率: ${stats.passRate}%`);
console.log(`总用例: ${stats.totalCases}`);
console.log(`已执行: ${stats.executed}`);
```

## 最佳实践

1. **使用 JSON Schema 验证**: 确保数据格式正确
2. **版本控制**: 使用 Git 管理 JSON 文件
3. **自动化集成**: 与 CI/CD 流程集成
4. **数据驱动**: 分离测试逻辑和测试数据
5. **可扩展性**: 使用自定义字段扩展
6. **文档化**: 添加详细的 description 字段
7. **标准化**: 统一命名规范和枚举值
