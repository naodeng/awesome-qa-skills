## 格式说明
适用于企业级缺陷跟踪系统、数据交换、工具集成。

## XSD Schema

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="bug">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="metadata" type="MetadataType"/>
        <xs:element name="reporter" type="PersonType"/>
        <xs:element name="assignee" type="AssigneeType"/>
        <xs:element name="dates" type="DatesType"/>
        <xs:element name="description" type="DescriptionType"/>
        <xs:element name="stepsToReproduce" type="StepsType"/>
        <xs:element name="expectedBehavior" type="BehaviorType"/>
        <xs:element name="actualBehavior" type="BehaviorType"/>
        <xs:element name="errorLogs" type="LogsType" minOccurs="0"/>
        <xs:element name="attachments" type="AttachmentsType" minOccurs="0"/>
        <xs:element name="technicalAnalysis" type="AnalysisType" minOccurs="0"/>
        <xs:element name="suggestedFixes" type="FixesType" minOccurs="0"/>
        <xs:element name="workaround" type="WorkaroundType" minOccurs="0"/>
        <xs:element name="verificationSteps" type="StepsType" minOccurs="0"/>
        <xs:element name="relatedIssues" type="IssuesType" minOccurs="0"/>
        <xs:element name="testCase" type="TestCaseType" minOccurs="0"/>
        <xs:element name="timeline" type="TimelineType" minOccurs="0"/>
        <xs:element name="comments" type="CommentsType" minOccurs="0"/>
        <xs:element name="metrics" type="MetricsType" minOccurs="0"/>
      </xs:sequence>
      <xs:attribute name="id" type="xs:string" use="required"/>
    </xs:complexType>
  </xs:element>

  <xs:simpleType name="StatusType">
    <xs:restriction base="xs:string">
      <xs:enumeration value="open"/>
      <xs:enumeration value="in_progress"/>
      <xs:enumeration value="fixed"/>
      <xs:enumeration value="verified"/>
      <xs:enumeration value="closed"/>
      <xs:enumeration value="wont_fix"/>
    </xs:restriction>
  </xs:simpleType>

  <xs:simpleType name="PriorityType">
    <xs:restriction base="xs:string">
      <xs:enumeration value="blocker"/>
      <xs:enumeration value="critical"/>
      <xs:enumeration value="high"/>
      <xs:enumeration value="medium"/>
      <xs:enumeration value="low"/>
    </xs:restriction>
  </xs:simpleType>
</xs:schema>
```

## 使用指南

### 1. 验证 XML
```bash
xmllint --schema bug-schema.xsd bug-report.xml
```

### 2. 转换为其他格式
```bash
# XML 转 JSON
xq . bug-report.xml > bug-report.json

# XML 转 HTML
xsltproc bug-to-html.xsl bug-report.xml > bug-report.html
```

### 3. 导入到工具
```bash
# 导入到 Jira
curl -X POST https://jira.example.com/api/import \
  -H "Content-Type: application/xml" \
  --data-binary @bug-report.xml
```

## 最佳实践

1. **使用 CDATA**: 对于包含特殊字符的内容
2. **命名空间**: 使用命名空间避免冲突
3. **验证**: 使用 XSD 验证 XML 格式
4. **编码**: 始终使用 UTF-8 编码
5. **格式化**: 保持良好的缩进
6. **注释**: 添加 XML 注释说明
7. **版本**: 在根元素添加版本属性
