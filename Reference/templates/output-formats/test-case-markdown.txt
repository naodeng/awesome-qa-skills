#### 预期结果
1. ✓ 页面跳转到首页 `https://example.com/dashboard`

4. ✓ 导航栏显示用户头像和用户名

#### 测试数据
```json

- **自动化状态**: 已自动化

### TC-004: 用户登录功能 - SQL 注入测试

**优先级**: Critical  
**类别**: Security  
**状态**: Not Run  
**负责人**: Security Team  
**标签**: `login` `security` `sql-injection` `owasp`

#### 前置条件
1. 应用程序已部署
2. 数据库连接正常

#### 测试步骤

测试以下 SQL 注入 payload:

1. **基础注入**
   - 用户名: `admin' OR '1'='1`
   - 密码: `' OR '1'='1`

2. **注释注入**
   - 用户名: `admin'--`
   - 密码: `anything`

3. **联合查询注入**
   - 用户名: `' UNION SELECT NULL, NULL--`
   - 密码: `anything`

4. **时间盲注**
   - 用户名: `admin' AND SLEEP(5)--`
   - 密码: `anything`

#### 预期结果
1. ✓ 所有注入尝试都应该失败
2. ✓ 显示通用错误消息（不泄露数据库信息）
3. ✓ 系统记录安全事件到日志
4. ✓ 触发安全警报（如果配置）
5. ✓ 响应时间正常（不受 SLEEP 影响）
6. ✓ 数据库查询使用参数化查询

#### 验证方法
```bash
# 检查日志
grep "SQL_INJECTION_ATTEMPT" /var/log/app/security.log

# 检查数据库查询
# 应该看到参数化查询，而不是字符串拼接
```

### TC-005: 用户登录功能 - 账号锁定机制

**优先级**: High  
**类别**: Security  
**状态**: Not Run  
**负责人**: QA Team  
**标签**: `login` `security` `account-lockout` `brute-force`

#### 前置条件
1. 用户已注册
2. 账号锁定策略: 5次失败后锁定30分钟
3. 账号当前未被锁定

#### 测试步骤
1. 使用有效用户名和错误密码尝试登录（第1次）
2. 重复步骤1（第2-5次）
3. 第6次尝试登录（使用正确密码）
4. 等待30分钟
5. 使用正确密码尝试登录

#### 预期结果

| 尝试次数 | 预期行为 |
|----------|----------|
| 1-5 | 显示 "用户名或密码错误" |
| 6 | 显示 "账号已被锁定，请30分钟后重试" |
| 6（正确密码） | 仍然显示锁定消息 |
| 30分钟后 | 登录成功 |

**额外验证**:
- ✓ 发送账号锁定通知邮件
- ✓ 记录锁定事件到审计日志
- ✓ 管理员可以手动解锁账号
- ✓ 锁定计数器在成功登录后重置

#### 测试数据
```json
{
  "username": "testuser@example.com",
  "correct_password": "Test@123",
  "wrong_password": "WrongPass123",
  "lockout_threshold": 5,
  "lockout_duration": 1800
}
```

## 使用指南

### 1. 创建新测试用例
```markdown
### TC-XXX: [功能名称] - [测试场景]

**优先级**: [Critical/High/Medium/Low]  
**类别**: [Functional/Negative/Security/Performance/UI]  
**状态**: [Not Run/Pass/Fail/Blocked]  
**负责人**: [团队/个人]  
**标签**: `tag1` `tag2` `tag3`

#### 前置条件
1. 条件1
2. 条件2

#### 测试步骤
1. 步骤1
2. 步骤2

#### 预期结果
1. ✓ 结果1
2. ✓ 结果2

#### 测试数据
\```json
{
  "key": "value"
}
\```
```

### 2. 更新测试状态
使用 emoji 或符号标记:
- ✓ Pass
- ✗ Fail
- ⊘ Blocked
- ○ Not Run
- ⟳ Retest

### 3. 链接缺陷
```markdown
**关联缺陷**: [BUG-123](https://jira.example.com/BUG-123)
```

### 4. 添加截图
```markdown
#### 实际结果
![错误截图](./screenshots/tc-001-error.png)
```

## 最佳实践

1. **使用清晰的标题**: 功能 + 场景
2. **保持步骤简洁**: 每个步骤一个操作
3. **使用代码块**: 测试数据、命令、配置
4. **添加表格**: 多场景测试、数据驱动
5. **使用 Checklist**: 验证点列表
6. **版本控制**: 使用 Git 管理测试用例
7. **Code Review**: 测试用例也需要评审
8. **持续更新**: 随功能变化更新用例

## 模板变体

### 简化版（快速测试）

**步骤**: 1. xxx 2. xxx 3. xxx

- 测试目标
