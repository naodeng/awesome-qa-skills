## 字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| Test Case ID | 唯一标识符，格式: TC-XXX | ✓ |
| Test Case Title | 测试用例标题 | ✓ |
| Priority | 优先级: Critical/High/Medium/Low | ✓ |
| Category | 类别: Functional/Negative/Security/Performance/UI | ✓ |
| Preconditions | 前置条件，多条用换行分隔 | ✓ |
| Test Steps | 测试步骤，编号列表 | ✓ |
| Expected Results | 预期结果，编号列表 | ✓ |
| Test Data | 测试数据，键值对格式 | - |
| Status | 状态: Not Run/Pass/Fail/Blocked/Skip | ✓ |
| Assigned To | 负责人 | - |
| Tags | 标签，分号分隔 | - |

## 导入步骤

### Excel
1. 打开 Excel
2. 选择 "数据" > "从文本/CSV"
3. 选择此 CSV 文件
4. 确认分隔符为逗号
5. 点击 "加载"

### Google Sheets
1. 打开 Google Sheets
2. 选择 "文件" > "导入"
3. 上传此 CSV 文件
4. 选择 "替换当前工作表"
5. 点击 "导入数据"

## 最佳实践

1. **ID 命名规范**
   - 使用前缀标识模块: LOGIN-001, CART-001
   - 保持连续编号
   - 预留编号空间用于扩展

2. **步骤编写**
   - 每个步骤一个操作
   - 使用清晰的动词: 点击、输入、选择
   - 包含具体的定位信息

3. **预期结果**
   - 与步骤一一对应
   - 可验证和可观察
   - 包含具体的值或状态

4. **标签使用**
   - 功能模块标签: login, checkout, search
   - 测试类型标签: smoke, regression, sanity
   - 优先级标签: p0, p1, p2

5. **数据管理**
   - 敏感数据使用占位符
   - 测试数据与生产数据隔离
   - 记录数据依赖关系

## 状态转换

```
Not Run → In Progress → Pass/Fail/Blocked
                      ↓
                   Retest
```

## 报告统计

使用 Excel 公式统计:
- 总用例数: `=COUNTA(A2:A100)`
- 通过率: `=COUNTIF(I2:I100,"Pass")/COUNTA(I2:I100)`
- 失败数: `=COUNTIF(I2:I100,"Fail")`
- 阻塞数: `=COUNTIF(I2:I100,"Blocked")`
