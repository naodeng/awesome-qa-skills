## 使用示例

### 1. 加载测试用例
```python
# Python
import yaml

with open('test-cases.yaml', 'r', encoding='utf-8') as f:
    test_suite = yaml.safe_load(f)

for test_case in test_suite['testSuite']['testCases']:
    print(f"执行测试: {test_case['title']}")
```

```javascript
// Node.js
const yaml = require('js-yaml');
const fs = require('fs');

const testSuite = yaml.load(fs.readFileSync('test-cases.yaml', 'utf8'));

testSuite.testSuite.testCases.forEach(testCase => {
  console.log(`执行测试: ${testCase.title}`);
});
```

### 2. 与 CI/CD 集成
```yaml
# .github/workflows/test.yml
name: Run Tests

on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Load test cases
        run: |
          yq eval '.testSuite.testCases[].id' test-cases.yaml
      
      - name: Run automated tests
        run: |
          npm test
```

### 3. 生成测试报告
```python
import yaml
from jinja2 import Template

# 加载测试用例
with open('test-cases.yaml', 'r') as f:
    data = yaml.safe_load(f)

# 生成 HTML 报告
template = Template('''
<html>
<head><title>测试报告</title></head>
<body>
  <h1>{{ suite.metadata.name }}</h1>
  <p>通过率: {{ suite.statistics.passRate }}%</p>
  <ul>
  {% for tc in suite.testCases %}
    <li>{{ tc.id }}: {{ tc.title }} - {{ tc.status }}</li>
  {% endfor %}
  </ul>
</body>
</html>
''')

html = template.render(suite=data['testSuite'])
with open('report.html', 'w') as f:
    f.write(html)
```

## 最佳实践

### 1. 使用锚点和别名减少重复
```yaml
# 定义锚点
common_preconditions: &common_preconditions
  - id: PRE-001
    description: 用户已注册
    required: true
  - id: PRE-002
    description: 应用程序已启动
    required: true

testCases:
  - id: TC-001
    preconditions: *common_preconditions  # 引用锚点
    
  - id: TC-002
    preconditions: *common_preconditions
```

### 2. 使用合并键扩展配置
```yaml
base_test: &base
  priority: medium
  category: functional
  automation:
    status: manual

testCases:
  - id: TC-001
    <<: *base  # 合并基础配置
    title: 测试用例 1
    priority: high  # 覆盖基础配置
```

### 3. 多文件组织
```yaml
# main.yaml
testSuite:
  metadata: !include metadata.yaml
  testCases: !include test-cases/*.yaml
```

### 4. 环境变量
```yaml
environment:
  url: ${TEST_URL:-https://staging.example.com}
  database: ${DB_NAME:-test_db}
```

### 5. 注释和文档
```yaml
testCase:
  id: TC-001
  title: 用户登录测试
  
  # 这是一个关键的安全测试
  # 必须在每次发布前执行
  priority: critical
  
  testSteps:
    - stepNumber: 1
      action: 打开登录页面
      # TODO: 添加页面加载性能检查
```

### 6. 验证 YAML 格式
```bash
# 使用 yamllint 验证
yamllint test-cases.yaml

# 使用 Python 验证
python -c "import yaml; yaml.safe_load(open('test-cases.yaml'))"
```

### 7. 转换为其他格式
```bash
# YAML 转 JSON
yq eval -o=json test-cases.yaml > test-cases.json

# YAML 转 XML
yq eval -o=xml test-cases.yaml > test-cases.xml
```

## 优势

1. **人类可读**: 比 JSON/XML 更易读
2. **简洁**: 更少的语法噪音
3. **注释支持**: 可以添加详细注释
4. **锚点和别名**: 减少重复
5. **多行字符串**: 支持多行文本
6. **类型推断**: 自动识别数据类型
7. **广泛支持**: 大多数编程语言都有 YAML 库
8. **CI/CD 友好**: 许多 CI/CD 工具原生支持 YAML
