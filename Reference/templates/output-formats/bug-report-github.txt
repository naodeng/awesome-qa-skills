# GitHub Issues 格式缺陷报告模板

## 格式说明
适用于 GitHub Issues 缺陷跟踪。

## 模板结构

```markdown
## 🐛 Bug Report

### Summary
账号锁定后未发送通知邮件

### Labels
`bug` `security` `email` `high-priority` `authentication`

### Environment
- **Version**: v1.2.3
- **Environment**: Staging
- **URL**: https://staging.example.com
- **Browser**: Chrome 120
- **OS**: Ubuntu 22.04 LTS
- **Database**: PostgreSQL 14.5

### Description
在用户账号因多次登录失败被锁定后，系统未能发送通知邮件给用户。这是一个安全问题，用户无法及时知道账号被锁定。

### Steps to Reproduce
1. 打开登录页面 `https://staging.example.com/login`
2. 使用有效用户名 `testuser@example.com` 和错误密码尝试登录
3. 重复步骤2共5次
4. 观察账号被锁定（显示锁定消息）
5. 检查用户邮箱 `testuser@example.com`
6. 查看邮件服务日志

### Expected Behavior
- ✓ 账号在第5次失败后被锁定
- ✓ 用户收到一封通知邮件
  - 主题: "账号安全警告"
  - 内容: 包含锁定原因、锁定时间、解锁时间
- ✓ 邮件在锁定后30秒内送达

### Actual Behavior
- ✓ 账号成功锁定
- ✗ 用户未收到任何邮件通知
- ✗ 邮件服务日志显示错误

### Error Messages
```
[2024-01-16 14:47:35] ERROR: SMTP connection timeout
[2024-01-16 14:47:35] ERROR: Failed to send email to testuser@example.com
[2024-01-16 14:47:35] ERROR: Retry attempt 1/3 failed
```

### Screenshots
![账号锁定界面](https://user-images.githubusercontent.com/xxx/screenshot-locked.png)
![邮件服务日志](https://user-images.githubusercontent.com/xxx/email-log.png)

### Additional Context
- **Test Case**: TC-005
- **Related PR**: #123
- **Database Record**: 锁定事件已正确记录到 `audit_logs` 表
- **Email Queue**: 队列为空，邮件未进入队列
- **SMTP Server**: 连接超时（5秒超时设置）

### Possible Root Cause
邮件服务配置的 SMTP 超时时间过短（5秒），在高负载情况下无法建立连接。建议增加到30秒。

### Workaround
手动通过管理后台发送通知邮件：
```bash
curl -X POST https://staging.example.com/admin/send-notification \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"userId": "12345", "type": "account_locked"}'
```

### Impact
- **Severity**: High
- **Affected Users**: 所有用户
- **Business Impact**: 用户体验差，安全通知缺失
- **Frequency**: 每次账号锁定都会发生

### Suggested Fix
1. 增加 SMTP 超时时间到30秒
2. 添加邮件发送重试机制（3次重试）
3. 添加邮件发送失败的备用通知方式（SMS）
4. 改进错误日志记录

### Related Issues
- Blocks #456 (发布测试)
- Related to #123 (邮件服务优化)
- Duplicate of #789 (可能重复)

### Checklist
- [x] 我已经搜索了现有的 issues
- [x] 我已经提供了完整的重现步骤
- [x] 我已经附加了相关的截图和日志
- [x] 我已经测试了最新版本
- [x] 我已经提供了环境信息

---

**Reporter**: @bob  
**Assignee**: @dev-team  
**Created**: 2024-01-16 15:00:00  
**Updated**: 2024-01-16 15:30:00
```

## 最佳实践

1. **使用模板**: 创建 issue 模板确保信息完整
2. **添加标签**: 使用标签分类和优先级
3. **关联 PR**: 链接相关的 Pull Request
4. **@提及**: 提及相关人员获得关注
5. **Markdown**: 使用 Markdown 格式化
6. **代码块**: 使用代码块展示日志和代码
7. **图片**: 使用图片展示问题
8. **Checklist**: 使用 checklist 确保完整性
