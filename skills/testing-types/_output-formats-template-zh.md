# 输出格式说明（中文模板）

本技能默认输出为 **Markdown**。若需要 **Excel**、**CSV** 或 **JSON** 格式，请在向 AI 提交需求时**在末尾明确说明**。

---

## 1. Markdown（默认）

不特别说明时，AI 按各测试类型 Standard-version 提示词中的 Markdown 模板输出，便于阅读与版本管理。

---

## 2. Excel 格式

**如何请求：** 在需求末尾加上一句，例如：

- 「请将上述测试用例/检查清单/报告以 **Excel 可粘贴的制表符分隔表格** 形式再输出一遍。」
- 「请用**制表符分隔的表格**输出，便于我复制到 Excel。」

**输出约定：**

- 第一行为表头（列名）。
- 列之间用 **Tab（制表符）** 分隔。
- 可直接复制整段粘贴到 Excel，即可自动分列。

**示例（测试用例表）：**

```
用例编号	用例标题	优先级	前置条件	步骤	预期结果
TC-01	登录-正确账号	P0	已安装应用	1.输入账号 2.点击登录	进入首页
TC-02	登录-错误密码	P1	已安装应用	1.输入错误密码 2.点击登录	提示密码错误
```

---

## 3. CSV 格式

**如何请求：** 在需求末尾加上，例如：

- 「请将结果以 **CSV 格式**（逗号分隔，首行为表头）输出。」
- 「请输出为 **CSV**，方便导入系统。」

**输出约定：**

- 第一行为列名（表头）。
- 列之间用英文逗号 `,` 分隔。
- 若单元格内容含逗号或换行，用双引号 `"` 包裹该单元格。

**示例：**

```csv
用例编号,用例标题,优先级,前置条件,步骤,预期结果
TC-01,登录-正确账号,P0,已安装应用,"1.输入账号 2.点击登录",进入首页
TC-02,登录-错误密码,P1,已安装应用,"1.输入错误密码 2.点击登录",提示密码错误
```

---

## 4. JSON 格式

**如何请求：** 在需求末尾加上，例如：

- 「请将上述内容以 **JSON** 形式输出。」
- 「请输出为 **JSON 数组**，便于程序解析。」

**输出约定：**

- 使用标准 JSON 语法。
- 表格类内容建议为对象数组，例如测试用例：`[{ "id": "TC-01", "title": "...", "priority": "P0", ... }, ...]`。
- 具体字段名由各测试类型的 Standard-version 输出结构决定，可在首次请求时说明「请用 JSON，并保持与 Markdown 中的信息一致」。

**示例（测试用例列表）：**

```json
{
  "testCases": [
    {
      "id": "TC-01",
      "title": "登录-正确账号",
      "priority": "P0",
      "preconditions": "已安装应用",
      "steps": "1.输入账号 2.点击登录",
      "expectedResult": "进入首页"
    },
    {
      "id": "TC-02",
      "title": "登录-错误密码",
      "priority": "P1",
      "preconditions": "已安装应用",
      "steps": "1.输入错误密码 2.点击登录",
      "expectedResult": "提示密码错误"
    }
  ]
}
```

---

使用时将本说明中的「如何请求」句子附加到你的需求后即可获得对应格式输出。
