## 📋 基本模板（1分钟）

```markdown
# 测试报告

## 1. 执行摘要
- 测试概况
- 主要发现
- 质量评估

## 2. 测试执行
- 执行统计
- 通过率

## 3. 缺陷分析
- 缺陷统计
- 严重程度分布

## 4. 风险与问题
- 当前风险
- 阻塞问题
```

## 🎯 快速示例（2分钟）

### 示例：Sprint 测试报告

```markdown
# Sprint 10 测试报告

**项目**: 电商平台
**Sprint**: Sprint 10
**测试周期**: 2024-01-22 ~ 2024-02-02
**报告人**: 张三

## 1. 执行摘要

### 测试概况
- 本 Sprint 完成 5 个用户故事测试
- 执行 120 个测试用例
- 发现 15 个缺陷
- 整体质量：良好

### 主要发现
- ✅ 核心功能稳定
- ⚠️  性能需要优化
- ❌ 发现 2 个 High 缺陷

### 质量评估
**评分**: 8/10
**建议**: 可以发布，但需修复 High 缺陷

## 2. 测试执行

### 执行统计
| 指标 | 数量 | 占比 |
|------|------|------|
| 计划用例 | 120 | 100% |
| 已执行 | 115 | 96% |
| 通过 | 100 | 87% |
| 失败 | 15 | 13% |
| 阻塞 | 5 | 4% |

### 通过率趋势
```
Week 1: 75%
Week 2: 87%
趋势: ↗️ 上升
```

## 3. 缺陷分析

### 缺陷统计
| 严重程度 | 新增 | 已修复 | 待修复 | 重开 |
|---------|------|--------|--------|------|
| Critical | 0 | 0 | 0 | 0 |
| High | 2 | 0 | 2 | 0 |
| Medium | 8 | 5 | 3 | 0 |
| Low | 5 | 3 | 2 | 0 |
| **总计** | **15** | **8** | **7** | **0** |

### 缺陷分布
- 功能缺陷: 60%
- 界面缺陷: 20%
- 性能缺陷: 20%

### 关键缺陷
1. **BUG-101** (High): 支付失败
   - 状态: 待修复
   - 影响: 阻塞发布

2. **BUG-105** (High): 数据丢失
   - 状态: 待修复
   - 影响: 阻塞发布

## 4. 风险与问题

### 当前风险
| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| High 缺陷未修复 | 高 | 中 | 加急修复 |
| 性能不达标 | 中 | 高 | 性能优化 |

### 阻塞问题
- 测试环境不稳定（已解决）
- 第三方 API 不可用（进行中）

## 5. 后续计划

### 待完成工作
- [ ] 修复 2 个 High 缺陷
- [ ] 回归测试
- [ ] 性能测试
- [ ] 发布前验证

### 改进建议
1. 提前进行性能测试
2. 增加自动化覆盖率
3. 加强代码审查

### 下个 Sprint 重点
- 性能优化验证
- 新功能测试
- 回归测试自动化

---

**结论**: 建议修复 High 缺陷后发布
```

## 🎨 报告类型（30秒）

### 日报（Daily）
```
- 今日完成
- 今日发现
- 明日计划
- 阻塞问题
```

### 周报（Weekly）
```
- 本周总结
- 测试进度
- 缺陷趋势
- 下周计划
```

### 发布报告（Release）
```
- 测试覆盖
- 质量评估
- 风险分析
- 发布建议
```

## ✅ 报告检查清单（30秒）

- [ ] 数据准确
- [ ] 图表清晰
- [ ] 结论明确
- [ ] 建议具体
- [ ] 格式统一
- [ ] 无错别字

## 💡 快速提示

### 关键指标
- 用例执行率
- 用例通过率
- 缺陷密度
- 缺陷修复率

### 可视化技巧
- 使用图表展示趋势
- 使用颜色标记状态
- 使用表格对比数据
- 使用图标增强可读性

### 写作技巧
1. **金字塔结构**: 结论先行
2. **数据说话**: 用数字支撑
3. **突出重点**: 标记关键信息
4. **提供建议**: 不只报告问题
