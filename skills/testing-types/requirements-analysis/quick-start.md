# 需求分析 - 5分钟快速上手

## 📋 基本模板（1分钟）

```markdown
# 需求分析报告

## 1. 需求概述
- 功能描述
- 业务目标
- 用户价值

## 2. 测试点识别
### 功能测试点
- 测试点1
- 测试点2

### 非功能测试点
- 性能要求
- 安全要求

## 3. 边界条件
- 输入边界
- 输出边界
- 时间边界

## 4. 测试优先级
- P0: 核心功能
- P1: 重要功能
- P2: 一般功能
```

## 🎯 快速示例（2分钟）

### 示例：登录功能需求分析

```markdown
# 登录功能需求分析

## 1. 需求概述
**功能**: 用户登录系统
**目标**: 验证用户身份，保护系统安全
**用户**: 注册用户、管理员

## 2. 测试点识别

### 功能测试点
- ✓ 正确用户名密码登录
- ✓ 错误用户名提示
- ✓ 错误密码提示
- ✓ 记住我功能
- ✓ 忘记密码链接

### 安全测试点
- ✓ SQL 注入防护
- ✓ XSS 攻击防护
- ✓ 暴力破解防护
- ✓ Session 管理

### 性能测试点
- ✓ 登录响应时间 < 2秒
- ✓ 并发登录支持

## 3. 边界条件

### 输入边界
| 字段 | 最小 | 最大 | 特殊值 |
|------|------|------|--------|
| 用户名 | 3字符 | 50字符 | 特殊字符 |
| 密码 | 8字符 | 20字符 | 特殊字符 |

### 异常场景
- 空用户名
- 空密码
- 超长输入
- 特殊字符
- SQL 语句
- 脚本代码

## 4. 测试优先级
- **P0**: 正常登录、安全防护
- **P1**: 错误提示、记住我
- **P2**: 忘记密码、UI 美化
```

## 🎨 分析方法（1分钟）

### 5W1H 分析法

```
What: 功能是什么
Who: 谁使用
When: 何时触发
Where: 在哪里使用
Why: 为什么需要
How: 如何实现
```

### 测试点分类

```
功能 → 正常/异常/边界
数据 → 有效/无效/边界
界面 → 布局/交互/响应
性能 → 速度/并发/资源
安全 → 认证/授权/加密
兼容 → 浏览器/设备/系统
```

## 💡 快速提示

### 测试点识别技巧
1. **正向思维**: 功能应该做什么
2. **逆向思维**: 功能不应该做什么
3. **边界思维**: 极限情况如何
4. **异常思维**: 出错时如何

### 常见遗漏点
- 权限验证
- 并发场景
- 超时处理
- 错误恢复
- 数据一致性
