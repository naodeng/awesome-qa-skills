# 缺陷上报 - 5分钟快速上手

本指南帮助你在 5 分钟内学会编写高质量的缺陷报告。

## 📋 基本模板（1分钟）

```markdown
# [模块] 简短描述问题

## 问题描述
用一两句话说明问题是什么

## 复现步骤
1. 第一步
2. 第二步
3. 第三步

## 预期结果
应该发生什么

## 实际结果
实际发生了什么

## 环境信息
- 操作系统：
- 浏览器/应用版本：
- 测试环境：

## 严重程度
[ ] Critical  [ ] High  [ ] Medium  [ ] Low

## 附件
- 截图/视频
- 日志文件
```

## 🎯 快速示例（2分钟）

### 示例 1：功能缺陷

```markdown
# [登录] 使用特殊字符密码无法登录

## 问题描述
当密码包含特殊字符（如 @、#、$）时，点击登录按钮无响应

## 复现步骤
1. 打开登录页面 https://example.com/login
2. 输入邮箱：test@example.com
3. 输入密码：Test@123
4. 点击"登录"按钮

## 预期结果
成功登录并跳转到首页

## 实际结果
- 登录按钮无响应
- 页面停留在登录页
- 控制台显示错误：`Invalid character in password`

## 环境信息
- 操作系统：macOS 13.5
- 浏览器：Chrome 120.0.6099.109
- 测试环境：Staging
- 应用版本：v2.3.1

## 严重程度
[x] High - 影响所有使用特殊字符密码的用户（约15%）

## 附件
- 截图：login-error.png
- 控制台日志：console-log.txt
```

### 示例 2：界面问题

```markdown
# [首页] 移动端导航菜单显示异常

## 问题描述
在 iPhone 12 上，导航菜单展开后无法关闭，遮挡页面内容

## 复现步骤
1. 使用 iPhone 12 打开应用
2. 点击左上角菜单图标
3. 菜单展开显示
4. 点击页面其他区域或菜单图标

## 预期结果
菜单应该收起，恢复正常页面显示

## 实际结果
菜单保持展开状态，无法关闭，遮挡页面内容

## 环境信息
- 设备：iPhone 12
- 操作系统：iOS 17.2
- 浏览器：Safari
- 屏幕尺寸：390x844

## 严重程度
[x] Medium - 影响移动端用户体验，但可通过刷新页面解决

## 附件
- 视频：menu-bug.mp4
- 截图：menu-stuck.png
```

## ✅ 质量检查清单（1分钟）

在提交缺陷报告前，检查以下项目：

### 必需项
- [ ] 标题清晰，包含模块名称
- [ ] 有详细的复现步骤
- [ ] 说明了预期结果和实际结果
- [ ] 包含环境信息
- [ ] 标记了严重程度

### 推荐项
- [ ] 添加了截图或视频
- [ ] 提供了日志文件
- [ ] 说明了影响范围
- [ ] 提供了变通方案（如果有）
- [ ] 检查了是否已有类似报告

## 🎨 标题编写技巧（30秒）

### 好的标题格式

```
[模块] 简短描述 + 关键条件
```

### 示例对比

| ❌ 不好的标题 | ✅ 好的标题 |
|-------------|-----------|
| 登录有问题 | [登录] 使用特殊字符密码无法登录 |
| Bug | [购物车] 删除商品后总价未更新 |
| 这个不工作 | [iOS] 应用在 iPhone 12 上启动崩溃 |
| 页面很慢 | [首页] 加载时间超过 5 秒（3G 网络） |

## 📸 附件最佳实践（30秒）

### 截图技巧
1. **标注关键区域**：使用箭头、圈圈标注问题位置
2. **包含上下文**：不要只截取错误信息，包含周围内容
3. **清晰可读**：确保文字和图标清晰可见

### 视频录制
1. **时长控制**：30秒-2分钟最佳
2. **操作清晰**：慢速操作，展示每个步骤
3. **文件大小**：压缩后上传（< 10MB）

### 日志文件
1. **相关部分**：只包含相关的日志行
2. **时间戳**：保留时间信息
3. **敏感信息**：移除密码、token 等

## 🔢 严重程度判断（30秒）

### 快速决策树

```
系统崩溃/数据丢失/安全问题？
  是 → Critical
  否 ↓

核心功能无法使用？
  是 → High
  否 ↓

有变通方案吗？
  无 → High
  有 ↓

严重影响用户体验？
  是 → Medium
  否 → Low
```

### 严重程度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| **Critical** | 系统崩溃、数据丢失、安全漏洞 | 应用闪退、数据库损坏、SQL 注入 |
| **High** | 核心功能无法使用，无变通方案 | 无法登录、无法支付、无法提交订单 |
| **Medium** | 功能可用但有明显问题，或有变通方案 | 按钮样式错误、提示信息不准确 |
| **Low** | 轻微问题，不影响使用 | 文字拼写错误、颜色不协调 |

## 🚀 常用工具

### 截图工具
- **macOS**: Cmd+Shift+4（系统自带）
- **Windows**: Win+Shift+S（Snipping Tool）
- **Chrome**: 开发者工具 → 截图功能
- **第三方**: Snagit、Lightshot

### 录屏工具
- **macOS**: QuickTime Player
- **Windows**: Xbox Game Bar (Win+G)
- **Chrome**: Loom 插件
- **跨平台**: OBS Studio

### 日志收集
- **浏览器**: F12 → Console → 右键 → Save as
- **移动端**: Xcode Console / Android Studio Logcat
- **服务器**: tail -f /var/log/app.log

## 💡 快速提示

### 复现步骤要点
1. **从头开始**：假设读者不了解系统
2. **具体数据**：使用具体的测试数据
3. **可重复**：任何人都能按步骤复现
4. **编号清晰**：使用数字编号

### 描述要点
1. **客观事实**：描述看到的，不是猜测的
2. **避免术语**：使用简单易懂的语言
3. **对比明确**：清楚说明预期 vs 实际
4. **量化数据**：用数字而非"很慢"、"很多"

## ⚠️ 常见错误

1. **标题太模糊**："有个 bug" → "[登录] 密码错误提示不显示"
2. **步骤不完整**：缺少前置条件、测试数据
3. **没有环境信息**：无法判断是否环境特定问题
4. **情绪化描述**："这个设计太烂了" → 客观描述问题
5. **一报告多问题**：每个问题单独报告，便于追踪
