## 🎯 快速示例（2分钟）

### 示例1：AI 生成测试数据

```markdown
# 提示词

请为登录功能生成测试数据，包括：
1. 有效用户名和密码（5组）
2. 无效用户名（边界值和异常值，10组）
3. 无效密码（边界值和异常值，10组）
4. SQL 注入测试数据（5组）
5. XSS 攻击测试数据（5组）

要求：
- 覆盖边界值（最小、最大、边界±1）
- 包含特殊字符
- 包含多语言字符
- 包含安全测试数据

# AI 输出

## 有效数据
1. user1@test.com / Test123!
2. admin@test.com / Admin@2024
...

## 无效用户名
1. "" (空字符串)
2. "ab" (少于3字符)
3. "a" * 51 (超过50字符)
...

## SQL 注入数据
1. ' OR '1'='1
2. admin'--
3. ' UNION SELECT * FROM users--
...
```

### 示例2：AI 分析缺陷根因

```markdown
# 提示词

分析以下缺陷的根因：

**缺陷描述**:
用户在购物车删除商品后，总价未更新

**复现步骤**:
1. 添加商品 A（¥100）
2. 添加商品 B（¥200）
3. 删除商品 A
4. 观察总价

**预期**: 总价 ¥200
**实际**: 总价 ¥300

**环境**: Chrome 120, macOS 13.5

# AI 分析

## 可能根因

### 1. 前端计算逻辑错误（概率：60%）
- 删除商品后未重新计算总价
- 建议检查：`calculateTotal()` 函数

### 2. 状态管理问题（概率：30%）
- 购物车状态未正确更新
- 建议检查：Redux/Vuex 状态

### 3. 事件监听问题（概率：10%）
- 删除事件未正确触发
- 建议检查：事件绑定

## 建议调查步骤
1. 检查浏览器控制台错误
2. 检查 `calculateTotal()` 函数
3. 检查状态管理代码
4. 添加日志追踪
```
