## 🎯 快速示例（2分钟）

### 示例：登录功能探索性测试

```markdown
# 登录功能探索性测试章程

**探索**: 登录功能的安全性和边界条件
**使用**: 手工测试 + Burp Suite
**发现**: 安全漏洞和用户体验问题

**时间盒**: 90 分钟
**测试人员**: 张三
**日期**: 2024-02-06

## 测试重点
- SQL 注入漏洞
- XSS 攻击
- 暴力破解防护
- Session 管理
- 错误提示

## 测试数据
- 特殊字符: `' OR '1'='1`
- SQL 语句: `admin'--`
- 脚本代码: `<script>alert('XSS')</script>`
- 超长字符串: 1000+ 字符

## 测试会话记录

### 10:00-10:30 SQL 注入测试
- 尝试: `' OR '1'='1`
- 结果: 被正确过滤 ✓
- 尝试: `admin'--`
- 结果: 被正确过滤 ✓

### 10:30-11:00 XSS 测试
- 尝试: `<script>alert('XSS')</script>`
- 结果: 未被过滤 ✗
- **发现缺陷**: BUG-001

### 11:00-11:30 暴力破解测试
- 尝试: 连续 10 次错误登录
- 结果: 无锁定机制 ✗
- **发现缺陷**: BUG-002

## 发现总结
- 缺陷数: 2
- 严重程度: High
- 建议: 增加 XSS 过滤和账号锁定
```

## 🎨 测试启发式（30秒）

### SFDPOT 启发式

```
S - Structure (结构): 测试系统架构
F - Function (功能): 测试功能实现
D - Data (数据): 测试数据处理
P - Platform (平台): 测试平台兼容
O - Operations (操作): 测试操作流程
T - Time (时间): 测试时间相关
```

### 测试旅程

```
商业区: 核心业务功能
历史区: 遗留功能
娱乐区: 有趣的功能
旅游区: 帮助文档
破坏区: 尝试破坏系统
```

## 💡 快速提示

### 探索技巧
1. **好奇心**: 问"如果...会怎样？"
2. **创造力**: 尝试意想不到的操作
3. **批判性**: 质疑一切假设
4. **系统性**: 有计划地探索

### 常用测试场景
- 边界值
- 特殊字符
- 并发操作
- 网络异常
- 权限越界
- 数据异常
