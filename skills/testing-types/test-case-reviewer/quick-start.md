# 测试用例评审 - 5分钟快速上手

## 📋 评审检查清单（1分钟）

```markdown
## 测试用例评审清单

### 基本信息 ✓
- [ ] 用例 ID 唯一
- [ ] 标题清晰
- [ ] 优先级标记
- [ ] 类型标记

### 前置条件 ✓
- [ ] 前置条件完整
- [ ] 前置条件可实现

### 测试步骤 ✓
- [ ] 步骤详细具体
- [ ] 步骤可重复

### 测试数据 ✓
- [ ] 数据具体明确

- [ ] 包含异常数据

### 预期结果 ✓
- [ ] 结果明确

### 覆盖率 ✓
- [ ] 正常场景
- [ ] 异常场景
- [ ] 边界条件
```

## 🎯 快速示例（2分钟）

### 示例：用例评审报告

```markdown
# 登录模块用例评审报告

**评审日期**: 2024-02-06
**评审人**: 张三
**用例数量**: 10
**评审范围**: 登录功能

## 评审总结
- **总体质量**: 良好
- **主要问题**: 边界值覆盖不足
- **改进建议**: 增加异常场景

## 问题统计
| 严重程度 | 数量 | 占比 |
|---------|------|------|
| Critical | 1 | 10% |
| High | 2 | 20% |
| Medium | 3 | 30% |
| Low | 2 | 20% |

## 详细问题

### Critical 问题
**TC-001**: 缺少 SQL 注入测试
- **影响**: 安全风险
- **建议**: 添加特殊字符测试用例

### High 问题
**TC-003**: 边界值测试不完整
- **影响**: 可能遗漏缺陷
- **建议**: 补充最小值、最大值、边界±1 测试

**TC-005**: 缺少并发登录测试
- **影响**: 性能风险
- **建议**: 添加并发场景用例

### Medium 问题
**TC-002**: 步骤描述不够具体
- **影响**: 执行困难
- **建议**: 详细说明每个操作

## 缺失场景
- Session 超时测试
- 密码复杂度验证
- 账号锁定机制测试

## 改进建议
1. 增加边界值测试（优先级：High）
2. 补充异常场景（优先级：High）
3. 优化用例描述（优先级：Medium）
4. 添加自动化标记（优先级：Low）
```

## 🎨 评审维度（30秒）

### 4C 评审法

```
Completeness (完整性)
- 需求覆盖率
- 场景覆盖率

Clarity (清晰度)
- 步骤明确
- 结果可验证

Correctness (正确性)
- 逻辑正确
- 数据准确

Consistency (一致性)
- 格式统一
- 术语一致
```

## ✅ 快速评审流程（30秒）

```
1. 浏览用例 (5分钟)
   - 了解整体结构
   - 识别明显问题

2. 详细检查 (15分钟)
   - 逐项检查清单
   - 记录问题

3. 覆盖率分析 (5分钟)
   - 识别缺失场景
   - 评估风险

4. 编写报告 (5分钟)
   - 总结问题
   - 提供建议
```

## 💡 快速提示

### 常见问题
- 步骤太粗略
- 缺少测试数据
- 预期结果模糊
- 用例有依赖
- 边界值遗漏

### 评审技巧
1. **换位思考**: 假设自己是执行者
2. **质疑一切**: 每个步骤都问"为什么"
3. **寻找遗漏**: 想象各种场景
4. **提供建议**: 不只指出问题，还要给方案
