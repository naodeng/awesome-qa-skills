# 测试用例编写 - 5分钟快速上手

本指南帮助你在 5 分钟内学会编写高质量的测试用例。

## 📋 基本模板（1分钟）

```markdown
## TC-001: 用例标题

**优先级**: P0 / P1 / P2 / P3
**类型**: 功能 / 界面 / 性能 / 安全

**前置条件**:
- 条件1
- 条件2

**测试步骤**:
1. 步骤1
2. 步骤2
3. 步骤3

**测试数据**:
- 数据项: 值

**预期结果**:
- 结果1
- 结果2

**实际结果**: [执行时填写]
**状态**: Pass / Fail
```

## 🎯 快速示例（2分钟）

### 示例 1：登录功能

```markdown
## TC-001: 使用有效凭据登录

**优先级**: P0
**类型**: 功能测试

**前置条件**:
- 用户已注册（test@example.com / Test123!）
- 用户状态正常（未锁定）

**测试步骤**:
1. 打开登录页面
2. 输入用户名：test@example.com
3. 输入密码：Test123!
4. 点击"登录"按钮

**预期结果**:
- 登录成功，跳转到首页
- 显示欢迎消息："欢迎，Test User"
- 右上角显示用户头像
- Session 创建成功

**实际结果**: [待执行]
**状态**: Not Run
```

### 示例 2：表单验证

```markdown
## TC-002: 邮箱格式验证

**优先级**: P1
**类型**: 功能测试

**前置条件**:
- 打开注册页面

**测试步骤**:
1. 在邮箱输入框输入：{test_email}
2. 点击其他区域（触发验证）
3. 观察验证提示

**测试数据**:
| 测试邮箱 | 预期结果 |
|---------|---------|
| test@example.com | ✓ 有效 |
| test@example | ✗ 无效 |
| @example.com | ✗ 无效 |
| test@.com | ✗ 无效 |
| test | ✗ 无效 |

**预期结果**:
- 有效邮箱：无错误提示，边框绿色
- 无效邮箱：显示"请输入有效的邮箱地址"，边框红色
```

## 🎨 测试设计方法（1分钟）

### 1. 等价类划分

将输入分为有效类和无效类：

```markdown
## 年龄输入（有效范围 18-65）

| 等价类 | 测试值 | 预期 |
|--------|--------|------|
| < 18 | 10, 17 | 拒绝 |
| 18-65 | 18, 30, 65 | 接受 |
| > 65 | 66, 100 | 拒绝 |
```

### 2. 边界值分析

测试边界值：

```markdown
## 年龄输入边界值

| 测试值 | 类型 | 预期 |
|--------|------|------|
| 17 | 边界-1 | 拒绝 |
| 18 | 下边界 | 接受 |
| 65 | 上边界 | 接受 |
| 66 | 边界+1 | 拒绝 |
```

### 3. 决策表

多条件组合：

```markdown
## 登录验证决策表

| 用户名 | 密码 | 结果 |
|--------|------|------|
| ✓ | ✓ | 登录成功 |
| ✓ | ✗ | 密码错误 |
| ✗ | ✓ | 用户名错误 |
| ✗ | ✗ | 都错误 |
```

## 🔢 优先级判断（30秒）

```
核心业务流程？
  是 → P0
  否 ↓

重要功能？
  是 → P1
  否 ↓

一般功能？
  是 → P2
  否 → P3
```

### 优先级定义

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0** | 核心流程，必须执行 | 登录、支付、下单 |
| **P1** | 重要功能，应该执行 | 搜索、筛选、收藏 |
| **P2** | 一般功能，可选执行 | 分享、评论、点赞 |
| **P3** | 边缘场景，时间充裕时执行 | 特殊字符、极端数据 |

## 📝 编写技巧（30秒）

### 步骤编写
1. **从头开始**：假设执行者不了解系统
2. **具体明确**：使用具体的数据和操作
3. **可重复**：任何人都能按步骤执行
4. **编号清晰**：使用数字编号

### 预期结果
1. **可验证**：结果明确，可以判断通过/失败
2. **完整**：包含所有需要验证的点
3. **具体**：不要使用"正确"、"正常"等模糊词

## 🚀 常用模式

### Given-When-Then 格式

```markdown
**Given（前置）**: 用户已登录
**When（操作）**: 点击"退出"按钮
**Then（结果）**: 
- 退出成功
- 跳转到登录页
- Session 清除
```

### 参数化用例

```markdown
## TC-001: 登录验证（参数化）

| ID | 用户名 | 密码 | 预期 |
|----|--------|------|------|
| 1 | valid | valid | 成功 |
| 2 | invalid | valid | 失败 |
| 3 | valid | invalid | 失败 |
| 4 | empty | valid | 失败 |
| 5 | valid | empty | 失败 |
```

## ⚠️ 常见错误

1. **步骤太粗**："登录系统" → "1. 打开登录页 2. 输入用户名 3. 输入密码 4. 点击登录"
2. **缺少数据**："输入用户名" → "输入用户名：test@example.com"
3. **结果模糊**："登录成功" → "跳转到首页，显示用户名，Session 创建"
4. **用例依赖**：TC-002 依赖 TC-001 → 每个用例独立
5. **没有优先级**：所有用例都一样重要 → 标记 P0/P1/P2/P3
