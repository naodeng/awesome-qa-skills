## 何时使用

- 需要把 API 测试结果落到 Bruno 结构或 Bruno 工作流里。
- 项目已经在用 Bruno，或希望输出直接适配 Bruno。

## 输出格式选项

默认使用 Markdown，除非需求明确要求其他格式。

## 如何使用

1. 打开 `prompts/api-test-bruno.md`，把它作为主提示词使用。
2. 补充真实项目上下文：范围、环境、限制、风险、依赖和期望产出。
3. 如果输入不完整，先给出可用初版，并标出信息缺口和假设。

## 常见误区

- 不要在范围和上下文都不清楚时直接使用。
- 不要把所有内容都当成同等重要。
- 不要跳过假设和信息缺口。

## 最佳实践

- 先从提示词正文出发，再补真正影响结果的上下文。
- 结果要按风险聚焦，而且能直接执行。
- 如果信息不全，先给可用初版，并把缺口标出来。
