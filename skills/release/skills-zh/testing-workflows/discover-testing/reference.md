# Testing Skill Routing Map

## Step 1: Choose Primary Skill Type

- Requirement understanding -> `requirements-analysis` / `requirements-analysis-en`
- Functional behavior validation -> `functional-testing` / `functional-testing-en`
- API contract and integration -> `api-testing` / `api-testing-en`
- Automation strategy or script design -> `automation-testing` / `automation-testing-en`
- Manual/exploratory sessions -> `manual-testing` / `manual-testing-en`
- Defect report authoring -> `bug-reporting` / `bug-reporting-en`
- Test case authoring -> `test-case-writing` / `test-case-writing-en`
- Test case quality review -> `test-case-reviewer` / `test-case-reviewer-en`
- Metrics and report outputs -> `test-reporting` / `test-reporting-en`
- Strategy and governance -> `test-strategy` / `test-strategy-en`
- Performance scope -> `performance-testing` / `performance-testing-en`
- Security scope -> `security-testing` / `security-testing-en`
- Accessibility scope -> `accessibility-testing` / `accessibility-testing-en`
- Mobile scope -> `mobile-testing` / `mobile-testing-en`
- AI-assisted workflows -> `ai-assisted-testing` / `ai-assisted-testing-en`

## Step 2: Choose Workflow Skill (if phase-based planning is needed)

- Daily execution cadence -> `daily-testing-workflow` / `daily-testing-workflow-en`
- Sprint-cycle coordination -> `sprint-testing-workflow` / `sprint-testing-workflow-en`
- Release readiness and go/no-go -> `release-testing-workflow` / `release-testing-workflow-en`

## Step 3: Add Supporting Skill

- Need report output -> add `test-reporting` / `test-reporting-en`
- Need stronger scope definition -> add `test-strategy` / `test-strategy-en`
- Need defect artifact quality -> add `bug-reporting` / `bug-reporting-en`
