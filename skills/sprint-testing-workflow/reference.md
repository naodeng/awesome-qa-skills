# 迭代测试工作流 — 提示词对照

执行各步骤时使用**本目录 `prompts/`** 下对应文件。下表为步骤与提示词文件对照。

| 步骤/阶段 | 提示词文件 | 用途 |
|-----------|------------|------|
| 第 1 天·规划 | test-strategy.md, requirements-analysis.md | 策略、范围、风险、测试数据 |
| 第 2–3 天·设置 | automation-testing.md, test-strategy.md, test-case-writing.md | 环境、CI/CD、早期用例 |
| 第 4–8 天·执行 | manual-testing.md, bug-reporting.md, automation-testing.md, api-testing.md | 探索、缺陷、自动化、API |
| 第 9–10 天·密集测试 | functional-testing.md, ai-assisted-testing.md, api-testing.md, accessibility-testing.md | 回归、E2E、智能选择、视觉 |
| 第 11 天·稳定化 | manual-testing.md | 探索性缺陷突击 |
| 第 12 天·评审 | test-reporting.md, test-strategy.md | 摘要、指标、仪表板 |
